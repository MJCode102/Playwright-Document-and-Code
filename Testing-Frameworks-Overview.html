Axios

What is Axios?

Axios is a popular, promise-based HTTP client library for JavaScript and Node.js. It is used to make HTTP requests to REST APIs.

Key Features

  • ✅ Promise-based API (works with async/await)
  • ✅ Automatic JSON data transformation
  • ✅ Request and response interceptors
  • ✅ Built-in request/response transformation
  • ✅ Client-side and server-side support
  • ✅ Automatic request body serialization
  • ✅ Request timeout support
  • ✅ Automatic error handling

Versions Used in Codebase

  • Version Range: 1.6.0 - 1.11.0
  • Most Common: 1.7.2

Usage Examples

// Basic GET request
const response = await axios.get('https://api.example.com/users');

// POST request with data
const response = await axios.post('https://api.example.com/users', {
  name: 'John Doe',
  email: 'john@example.com'
});

// With configuration
const response = await axios({
  method: 'POST',
  url: '/api/users',
  baseURL: 'https://api.example.com',
  headers: {
    'Authorization': 'Bearer token123',
    'Content-Type': 'application/json'
  },
  data: { name: 'John Doe' },
  timeout: 5000
});

Where Axios is Used

  • API automation frameworks
  • Test suites for REST API endpoints
  • Backend services (NestJS applications)
  • Integration with Jest, Mocha, and other testing frameworks

Jest

What is Jest?

Jest is a JavaScript testing framework developed by Facebook. It provides a complete testing solution with built-in assertions, mocking, code coverage, and test running capabilities.

Key Features

  • ✅ Built-in assertion library (no need for separate assertion library)
  • ✅ Powerful mocking capabilities
  • ✅ Code coverage reports (built-in)
  • ✅ Snapshot testing
  • ✅ Parallel test execution
  • ✅ Watch mode for development
  • ✅ Excellent async/await support
  • ✅ TypeScript support (with ts-jest)

Versions Used in Codebase

  • Version: 29.7.0
  • TypeScript Support: ts-jest 29.1.2

Jest Configuration Example

{
  "jest": {
    "testEnvironment": "node",
    "testMatch": ["**/tests/**/*.spec.js"],
    "testTimeout": 60000,
    "collectCoverageFrom": [
      "framework/**/*.js",
      "utility/**/*.js"
    ]
  }
}

Common Jest Assertions

// Equality
expect(value).toBe(4);
expect(value).toEqual({ name: 'John' });

// Truthiness
expect(value).toBeTruthy();
expect(value).toBeFalsy();

// Arrays
expect(array).toContain(item);
expect(array).toHaveLength(3);

// Async
await expect(promise).resolves.toBe(value);
await expect(promise).rejects.toThrow();

Chai

What is Chai?

Chai is a BDD/TDD assertion library for Node.js and browsers. It provides three assertion styles: assert, expect, and should.

Key Features

  • ✅ Three assertion styles (assert, expect, should)
  • ✅ Chainable language constructs
  • ✅ Extensive plugin ecosystem
  • ✅ Works with any test runner (Mocha, Jest, etc.)
  • ✅ Browser and Node.js support

Versions Used in Codebase

  • Version: 4.4.1

Chai Assertion Styles

Assert Style

const assert = require('chai').assert;
assert.equal(value, 4);
assert.typeOf(value, 'string');
assert.property(obj, 'name');

Expect Style (Most Common)

const expect = require('chai').expect;
expect(value).to.equal(4);
expect(value).to.be.a('string');
expect(obj).to.have.property('name');
expect(array).to.include(3);

Should Style

const should = require('chai').should();
value.should.equal(4);
value.should.be.a('string');
obj.should.have.property('name');

Where Chai is Used

  • Framework Approach 1 - Mocha + Chai + Axios combinations
  • Various Mocha-based test suites
  • Legacy test frameworks

Mocha

What is Mocha?

Mocha is a feature-rich JavaScript test framework running on Node.js and in the browser. It provides a flexible testing structure and works with any assertion library.

Key Features

  • ✅ Flexible and simple
  • ✅ Works with any assertion library (Chai, Should.js, etc.)
  • ✅ Sequential test execution (useful for dependent tests)
  • ✅ Rich hooks (before, after, beforeEach, afterEach)
  • ✅ Async support (callbacks, promises, async/await)
  • ✅ Multiple reporters (spec, dot, json, etc.)
  • ✅ Mature and stable ecosystem

Versions Used in Codebase

  • Version Range: 10.2.0 - 10.7.3
  • Most Common: 10.7.3

Mocha Test Structure

describe('User API Tests', function() {
  before(function() {
    // Runs once before all tests
  });

  beforeEach(function() {
    // Runs before each test
  });

  it('should create a user', async function() {
    const response = await axios.post('/api/users', userData);
    expect(response.status).to.equal(201);
  });

  afterEach(function() {
    // Runs after each test
  });

  after(function() {
    // Runs once after all tests
  });
});

Mocha Reporters

  • spec - Hierarchical spec list
  • dot - Dots for passing tests
  • json - JSON output
  • mochawesome - HTML reports (used in rcs-test project)

Framework Comparison

DTC-32 Framework Approaches

Based on the DTC-32 project, three framework approaches were implemented:

Approach Framework Stack Test Runner Assertions Language Status
Approach 1 Mocha + Chai + Axios Mocha Chai JavaScript ✅ PASSED
Approach 2 Jest + Axios Jest Jest (built-in) JavaScript ✅ PASSED
Approach 3 TypeScript + Jest + Axios Jest Jest (built-in) TypeScript ✅ PASSED

Detailed Comparison

Approach 1: Mocha + Chai + Axios

Pros:

  • ✅ Flexible (works with any assertion library)
  • ✅ Sequential execution (good for dependent tests)
  • ✅ Mature and stable
  • ✅ Rich hooks support

Cons:

  • ❌ Requires separate assertion library (Chai)
  • ❌ Slower execution (sequential)
  • ❌ No built-in mocking

Best For:

  • Legacy projects
  • Tests with dependencies
  • Teams familiar with Mocha

Approach 2: Jest + Axios

Pros:

  • ✅ Built-in assertions (no Chai needed)
  • ✅ Powerful mocking capabilities
  • ✅ Faster execution (parallel)
  • ✅ Built-in code coverage
  • ✅ Better developer experience

Cons:

  • ❌ Less flexible than Mocha
  • ❌ Parallel execution can cause issues with shared state

Best For:

  • New projects
  • Teams wanting modern tooling
  • Projects needing code coverage

Approach 3: TypeScript + Jest + Axios

Pros:

  • ✅ All benefits of Jest + Axios
  • ✅ Type safety
  • ✅ Better IDE support
  • ✅ Compile-time error checking
  • ✅ Better refactoring support

Cons:

  • ❌ Requires TypeScript knowledge
  • ❌ Additional compilation step
  • ❌ More complex setup

Best For:

  • Large projects
  • Teams using TypeScript
  • Projects requiring type safety

Common Patterns

Jest + Axios Pattern

const axios = require('axios');

describe('API Tests', () => {
  test('should get users', async () => {
    const response = await axios.get('/api/users');
    expect(response.status).toBe(200);
    expect(response.data).toHaveLength(10);
  });
});

Mocha + Chai + Axios Pattern

const axios = require('axios');
const expect = require('chai').expect;

describe('API Tests', function() {
  it('should get users', async function() {
    const response = await axios.get('/api/users');
    expect(response.status).to.equal(200);
    expect(response.data).to.have.lengthOf(10);
  });
});

Summary

Axios

  • Purpose: HTTP client for API requests
  • Versions: 1.6.0 - 1.11.0
  • Used with: Jest, Mocha, NestJS

Jest

  • Purpose: Complete testing framework
  • Version: 29.7.0
  • Features: Built-in assertions, mocking, coverage
  • Best with: Axios for API testing

Chai

  • Purpose: Assertion library
  • Version: 4.4.1
  • Used with: Mocha (primarily)

Mocha

  • Purpose: Test runner framework
  • Versions: 10.2.0 - 10.7.3
  • Used with: Chai + Axios
  • Features: Flexible, sequential execution

Recommendation

For new projects, Jest + Axios is recommended because:

  • Modern and actively maintained
  • Built-in features (assertions, mocking, coverage)
  • Better developer experience
  • Faster execution
  • TypeScript support

For existing Mocha projects, continue using Mocha + Chai + Axios as it's stable and well-established.
