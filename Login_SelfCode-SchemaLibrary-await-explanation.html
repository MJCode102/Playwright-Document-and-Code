Playwright → Selenium (Java) Conversion

This document explains how each important await in Login_SelfCode-SchemaLibrary.spec.js maps to Selenium with Java in LoginSelfCodeSchemaLibraryTest.java, focusing on waits and actions. Use this as a talking guide in your interview.

1. Login Flow

Playwright (JS) Selenium (Java) What to Explain
await page.goto(url) driver.get(url); In Playwright, await waits for navigation to finish. In Selenium, get() is synchronous but we usually add explicit waits for elements we need next instead of sleeping.
await page.getByPlaceholder("Email address").fill(...) wait wait.until(visibilityOfElementLocated(...));
action emailInput.sendKeys(...); 		Playwright auto-waits for the element to be ready before typing. In Selenium we explicitly wait using WebDriverWait + ExpectedConditions.visibilityOfElementLocated, then call sendKeys().
await page.click("button:has-text('Next')") WebElement nextButton = driver.findElement(...);
nextButton.click(); 		Playwright combines waiting + click. In Selenium, you typically either do a wait.until(elementToBeClickable(...)) or rely on the DOM being stable and then call click(). In the sample we show both direct findElement and waited actions.
await page.getByPlaceholder("Your password").fill(...)
await page.waitForTimeout(1000) 		WebElement passwordInput = wait.until(visibilityOfElementLocated(...));
passwordInput.sendKeys(password); 		The waitForTimeout is a hard sleep. In Selenium, instead of fixed sleeps, we prefer explicit waits that stop as soon as the element is ready. This is more reliable and faster.
await page.click("button:has-text('Sign on')") WebElement signOnButton = driver.findElement(...);
signOnButton.click(); 		Same idea: Playwright waits internally; Selenium needs either an explicit wait for clickability or relies on the previous waits.

6. High-Level Talking Points for the Interview

  • Auto-wait vs Explicit Waits – Playwright automatically waits for elements; in Selenium we must explicitly use WebDriverWait with conditions like visibilityOfElementLocated and elementToBeClickable.
  • Replacing Hard Sleeps – page.waitForTimeout() in Playwright should be replaced with semantic waits in Selenium (wait for specific elements or states, not time).
  • Locator Strategies – Playwright’s getByTestId, getByRole, and text locators map to Selenium By.cssSelector, By.id, and By.xpath with similar semantics.
  • Structure – The asynchronous async/await pattern in JavaScript becomes synchronous-looking Java code, but the “waiting” responsibility is moved into WebDriverWait calls.
